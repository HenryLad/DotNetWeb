Cross-Platform Web App with ASP.NET Core

Creating a Cross-Platform Web App with ASP.NET Core

Open a terminal or command prompt and run the following commands:

        
bash
dotnet new web -n MyWebApp
cd MyWebApp

Open the project in your chosen IDE.

Build and run the application:

        
bash
dotnet build
dotnet run

Open your web browser and navigate to http://localhost:5000 to see your web app.

Building a Cross-Platform Service with ASP.NET Core

Open a terminal or command prompt and run the following commands:

        
bash
dotnet new webapi -n MyWebService
cd MyWebService

Open the project in your chosen IDE.

Build and run the service:

        
bash
dotnet build
dotnet run

The service will be accessible at http://localhost:5000/api/values.

Containerizing with Docker

Create a Dockerfile in the project directory:

        
FROM mcr.microsoft.com/dotnet/aspnet:5.0
WORKDIR /app
COPY bin/Debug/net5.0/publish/ .
ENTRYPOINT ["dotnet", "MyWebApp.dll"]

Make sure to adjust the COPY command based on your project structure.

Open a terminal and navigate to the project directory.

Build the Docker image:

        
bash
docker build -t mywebapp .

Run the Docker container:

        
bash
docker run -p 8080:80 mywebapp

Access your web app in a browser at http://localhost:8080.

Conclusion

You've successfully created cross-platform web applications and services using C# and .NET Core. You can deploy these applications on macOS, Windows, Linux, or containerize them with Docker for easy distribution. Happy coding!
